# Designing a Robust MEA-Based Post-Combustion Carbon Capture Process with Capture Rate Guarantees

## Introduction

This directory contains the source code accompanying the paper:

> J.A.F. Sherman, A.G. Ostace, D.A. Allan, C.E. Gounaris,
"Robust Optimization of an MEA-Based Post-Combustion Carbon Capture
Process at High Carbon Capture Rates". Submitted for publication, 2026.

If you find the contents of this directory useful for your research,
please consider citing the paper.

This work was developed as part of a research project 
funded by the Department of Energy’s Carbon Capture Simulation for
Industry Impact (CCSI<sup>2</sup>) program.


## Setup

<a name="prerequisites"></a>
### Prerequisites

In advance of successfully running the code in the present directory,
you will need working distributions of:

1. [Python](https://docs.python.org/3/using/index.html) 3.9, 3.10, 3.11, or 3.12.

2. [GAMS](https://www.gams.com/49/docs/UG_MAIN.html#UG_INSTALL)
   (49 or later recommended).

3. [IPOPT](https://coin-or.github.io/Ipopt/INSTALL.html),
   with access to
   [HSL linear solvers](https://licences.stfc.ac.uk/product/coin-hsl).
   Alternatively, IPOPT can be installed
   [as an extension of the IDAES-pse software package](
      https://idaes-pse.readthedocs.io/en/stable/tutorials/getting_started/index.html
   )
   after [installing the Python dependencies](#dependencies)
   for the present codebase.

4. LaTeX, which we recommend setting up by
   [installing TeX Live](https://www.tug.org/texlive/).
   If TeX Live is used, then we require at minimum a basic installation with
   the additional packages/package collections
   `collection-latexrecommended`,
   `collection-fontsrecommended`,
   `type1cm`,
   `sfmath`,
   and `dvipng`,
   all of which can be installed using the TeX Live package manager
   (`tlmgr`).
   The LaTeX distribution is required for successfully
   rendering any and all plots generated by the present codebase
   using the Python package [Matplotlib](https://matplotlib.org/).
   For more information on rendering text with LaTeX in Matplotlib, see the
   [corresponding Matplotlib documentation page](
      https://matplotlib.org/stable/users/explain/text/usetex.html
   ).

Note that the published results were produced with Python 3.12.11,
GAMS 49.6.1, IPOPT 3.13.2/MA57, and are tied to
[commit 9aac556](
	https://github.com/IDAES/publications/tree/9aac556aca25ed2dc58245452f402cc51f774e60
)
of the present repository.


### Instructions

In a UNIX terminal, you may set up the present package as follows:

1. Clone the online repository containing this directory
   to your desired location, which we refer to as `$REPO_DIR`:

   ```sh
   git clone https://github.com/IDAES/publications.git
   ```

2. Change your working directory to the `sherman_et_al_2026` subdirectory
   of the cloned repository:

   ```sh
   cd $REPO_DIR/sherman_et_al_2026
   ```

   We hereby use `$MEA_DIR` to denote the location of this subdirectory
   on your file system.

<a name="dependencies"></a>

3. In a Python virtual environment, install the dependencies
   that were used with the present package:

   ```sh
   pip install -r requirements_pyros.txt
   ```

   Note that the [IDAES-pse](https://github.com/IDAES/idaes-pse)
   and [Pyomo](https://github.com/Pyomo/pyomo)
   software packages are automatically installed as dependencies.
   If an IPOPT distribution is not already available on your
   system, you may now install IPOPT as an extension of the
   IDAES-pse package
   (using ``idaes get-extensions``)
   according to the
   [IDAES installation guide](
      https://idaes-pse.readthedocs.io/en/stable/tutorials/getting_started/index.html
   ).
   In most cases, the IPOPT installation provided by IDAES
   [includes HSL linear solvers](
      https://idaes-pse.readthedocs.io/en/stable/tutorials/getting_started/binaries.html
   ).

4. (Optional, but recommended) Test your setup:

   ```sh
   pytest -v test_mea_fs_costing_pyros.py
   ```

   Note that the tests may take several minutes to run,
   depending on your system.
   Some tests may fail if the [prerequisites](#prerequisites) are not met.

## Usage

In the examples that follow,
set your working directory to `$MEA_DIR`,
and replace `$RESULTS_DIR`
with a path to a directory of your choice for storing
the results of computational studies.
If `$RESULTS_DIR` does not exist,
then it is automatically created,
provided that its parent directories exist.


### Running Deterministic Optimization Cases

For a given set of CO<sub>2</sub> capture targets, say 90%, 95%, and 99%,
the flowsheet model can be solved deterministically as follows:

```sh
python run_combined_flowsheet_pyros_econ_obj.py --workflow deterministic --with-postsolve-costing True --co2-capture-targets 90 95 99 --results-dir $RESULTS_DIR
```

A collection of `*.csv` and `*.png` files containing the
results is stored in `$RESULTS_DIR/deterministic/`;
file names are descriptive.
In particular, key outputs of the model and the solver(s)
at the optimal solution(s)
obtained are recorded in the file
`$RESULTS_DIR/deterministic/deterministic_results.csv`,
the colums of which are documented in the file
`mea_solve_results_column.csv`.
The `*.png` images visualize the responses of several key outputs
to the CO<sub>2</sub> capture target.


### Running Robust Optimization Cases

For a given capture target and uncertainty set confidence level,
the flowsheet model can be solved to robust optimality as follows:

```sh
python run_combined_flowsheet_pyros_econ_obj.py --workflow pyros --with-postsolve-costing True --co2-capture-targets 95 --pyros-confidence-levels 99 --results-dir $RESULTS_DIR
```

A collection of `*.csv` and `*.png` files recording the results,
with the recourse variables evaluated according to the PyROS decision rules
subject to the nominal uncertain parameter realization,
is stored in `$RESULTS_DIR/pyros/95pt0capture_99pt0_conf/`;
file names are descriptive.
In particular, key outputs of the model at the solution obtained
are stored in the file entitled
`$RESULTS_DIR/pyros/95pt0capture_99pt0_conf/solve_results.csv`,
the column labels of which are documented in the file
`mea_solve_results_column.csv`.


### Compiling the Results of Case Runs

After solving a set of deterministic and/or robust optimization instances,
the results can be visualized and compiled into `*.csv` files,
as follows:

```sh
python run_combined_flowsheet_pyros_econ_obj.py --workflow solve_results --results-dir $RESULTS_DIR
```

The processed results are stored in
`$RESULTS_DIR/solve_results_analysis/`,
which contains several descriptively named `*.csv` files,
PNG/PDF files, and subdirectories containing other such files.
In particular, key outputs of the model solutions obtained are
compiled into the file
`$RESULTS_DIR/solve_results_analysis/pyros_results_full.csv`,
the index labels of which are
documented in the file `mea_solve_results_column.csv`.
All results are based on evaluation of the recourse variables
subject to the PyROS decision rules under the nominal
uncertain parameter realization.

For more information on using the `run_combined_flowsheet_pyros_econ_obj.py`
script, run `run_combined_flowsheet_pyros_econ_obj.py --help`
in your terminal.


### Running a Full Computational Study

In advance of running a full computational study, you will need to
empty or delete ``$RESULTS_DIR`` before proceeding and:

1. Ensure that you have a working distribution of
   [GNU Screen (`screen`)](https://www.gnu.org/software/screen/) installed
   on your system
2. Ensure that there are no active `screen` sessions with name containing
   "pyros\_mea" (check with `screen -ls`)
3. Add the execute permission to the script
   `./pyros_flowsheet_runs.sh`.
   This can be achieved by running in your terminal the command:

   ```sh
   chmod u+x pyros_flowsheet_runs.sh
   ```

4. Create a new `screen` session with name "mea\_main", as follows:

   ```sh
   screen -S mea_main
   ```

Within the new `screen` session,
a full computational study, in which the model is solved
for several CO<sub>2</sub> capture targets and uncertainty set
confidence levels, can be performed by running the command:

```sh
./pyros_flowsheet_runs.sh -d $RESULTS_DIR -t $CO2_CAPTURE_TARGETS -l $CONFIDENCE_LEVELS
```

in which `$CO2_CAPTURE_TARGETS` is to be replaced with
a desired sequence of capture targets
(e.g., `'90 96 99'`)
and `$CONFIDENCE_LEVELS` is to be replaced with
a desired sequence of
ellipsoidal uncertainty set confidence levels for robust optimization
(e.g., `'90 95 99'`).

> [!NOTE]
> The model solves performed by the script `pyros_flowsheet_runs.sh`
> are carried out in parallel, with one CPU core
> of your machine essentially reserved for robust optimization
> at each CO<sub>2</sub> capture target
> and one CPU core reserved for sequentially solving the model
> deterministically subject to all targets.
> Therefore, you should ensure that there are more available
> CPU cores on your system than there are CO<sub>2</sub>
> capture targets that you have specified.

Successful execution of the script `pyros_flowsheet_runs.sh` will:

- Solve the model deterministically, subject to each of the
  CO<sub>2</sub> capture targets listed in `$CO2_CAPTURE_TARGETS`,
  storing the results in `$RESULTS_DIR/deterministic/`
- Solve the model robustly, subject to each of the
  CO<sub>2</sub> capture targets listed in `$CO2_CAPTURE_TARGETS`
  and each of the confidence levels listed in `$CONFIDENCE_LEVELS`,
  storing the results in `$RESULTS_DIR/pyros/`
- Compile the results and store them in
  `$RESULTS_DIR/solve_results_analysis/`

The full computational study may take several hours to run.

For more information on configuring the study,
run ``./pyros_flowsheet_runs.sh -h`` in your terminal.
